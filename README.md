# Day 7: SQL Filtering & Sorting Practice — Hospital Patients Dataset

## 🏥 Project Overview
This repository contains a series of SQL queries and data interpretations focused on hospital admission scenarios. The goal of this project is to extract actionable medical, financial, and operational insights from raw healthcare data using advanced filtering and sorting techniques.

## 🛠️ Key SQL Skills Demonstrated
* **Complex Filtering:** Utilizing `AND`, `OR`, and `NOT` logic to isolate specific patient demographics and unit statuses.
* **Range & Pattern Matching:** Applying `BETWEEN`, `IN`, and wildcard `LIKE` operators for targeted data retrieval (e.g., specific age brackets, billing ranges, and diagnosis string variations).
* **Data Sorting & Limiting:** Using `ORDER BY` and `LIMIT` to accurately generate priority lists for financial audits and recent admission dashboards.
* **Data Interpretation:** Translating raw query results into plain-English operational insights for non-technical stakeholders (e.g., medical directors, finance teams).

## 📊 Scenarios Explored
1. **ICU Critical Care Report:** Identified high-cost critical care patients and analyzed physician workload bottlenecks.
2. **Senior Citizen Care Analysis:** Filtered high-risk demographic data to prioritize severe cases across specialized departments.
3. **High-Value Billing Review:** Flagged targeted financial outliers for insurance auditing, revealing clear correlations between departments and room types.
4. **Diagnosis Pattern Search:** Used string matching to isolate oncology cases, identifying severity and cost trends based on patient age and cancer stages.
5. **Recent Admissions Dashboard:** Built a time-bounded operational view of stable/discharged patients within a specific quarterly timeframe.
